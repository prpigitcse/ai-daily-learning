The Theory

Quantization is the process of mapping continuous (floating-point) values to a smaller set of discrete levels. In the context of TinyML, this typically means converting 32-bit floating-point weights ($FP32$) to 8-bit integers ($INT8$).

The goal is to reduce the memory footprint and increase inference speed on microcontrollers without significant loss in model accuracy. By reducing precision, we drastically lower the energy cost of memory access, which is often the primary bottleneck in edge devices.

The Math

The basic uniform affine quantization transformation is defined as:

$$Q(r) = \text{round}\left(\frac{r}{S} + Z\right)$$
$r$

The real-valued input (floating point weight or activation).

$S$

The scale factor (a positive floating point number).

$Z$

The zero-point (an integer representing the real value 0).

To recover the approximate real value ($r \approx \hat{r}$), we use: $$\hat{r} = S(Q - Z)$$

The Code

import numpy as np

def quantize_tensor(r, num_bits=8):
    # Determine the range
    r_min, r_max = r.min(), r.max()
    
    # Calculate scale (S)
    q_min, q_max = 0, (2**num_bits) - 1
    S = (r_max - r_min) / (q_max - q_min)
    
    # Calculate zero-point (Z)
    Z = q_min - np.round(r_min / S)
    Z = np.clip(Z, q_min, q_max)
    
    # Quantize
    q = np.round(r / S + Z)
    return np.clip(q, q_min, q_max).astype(np.uint8), S, Z

Logic Breakdown

1. Min-Max Calculation

We first observe the dynamic range of the layer's weights to ensure we map the scale factor across the entire spread of values.

2. Scale Factor (S)

The scale maps the real range to the available bit-depth (e.g., 255 levels for 8-bit).

3. Zero-Point Adjustment

Crucial for asymmetric distributions; it ensures that the value 0.0 in the real world maps exactly to an integer in the quantized space.
